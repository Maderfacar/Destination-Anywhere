二、協作開發完整流程（從 0 到 100）
總原則：

嚴格遵循「規則先行、任務可追蹤、小步驟、強制 commit」
每個階段結束必須通過 Stage Gate（關卡審核）才能進入下一階段
Agent 每次回應必須遵守 agent-protocols.md 中的思考格式
每完成一個子任務，必須立即更新 tasks.md 並 commit
同一個錯誤連續出現 2 次 → 立即停止並要求人類介入（Stop Loss）

Stage 0：需求與規則最終確認（The Constitution）

Agent 完整閱讀所有核心 MD 文件（特別是 prd.md、tech-stack.md、roadmap.md）。
輸出「一致性檢查報告」（檢查各文件之間是否有衝突）。
根據 prd.md 產生 tasks.md 初稿（將所有功能細分成可執行的具體任務）。
你（人類）審核並確認：
prd.md 是否完整
tech-stack.md 中的已知套件與「待確認清單」
tasks.md 的細分是否合理

Stage Gate：你親自批准所有規則文件與 tasks.md 初稿後，才允許進入 Stage 1。

Stage 1：環境初始化與規則設定（The Law）

建立專案、初始化 Git。
建立所有核心 MD 文件。
安裝目前已確定的套件與工具（包含已知的外部 API SDK）。
在 tech-stack.md 中明確標註「待確認」的插件或 API（例如 i18n、多語系、Line Bot SDK 等）。
設定 .cursorrules、TypeScript、Tailwind 等基礎環境。
Stage Gate：npm run build + npm run lint 通過，tasks.md 已 commit。

Stage 2：基礎原子組件開發（The Atoms）

根據 style-guide.md 只開發純 UI 元件（Button、Input、Card、Modal 等）。
禁止加入任何業務邏輯。
目標：視覺風格完全統一。
Stage Gate：元件可在 Storybook 或展示頁正常顯示 + build 通過。

Stage 3：佈局與靜態頁面（The Shell）

按照 roadmap.md 與 folder-structure.md 搭建 Header/Footer、路由結構、靜態頁面框架。
目標：看到網站完整骨架，但暫無真實資料。
Stage Gate：所有路由可正常切換 + build 通過。

Stage 4：邏輯實作與狀態管理（The Logic）

實作表單驗證、按鈕互動、頁面間資料傳遞、狀態管理。
目標：網頁開始「動起來」。
若發生鬼打牆，立即觸發 agent-protocols.md 的停損機制。
Stage Gate：所有互動正常、無明顯 console error。

Stage 5：資料串接與持久化（The Soul）

串接真實 API / Database。
處理 Loading、錯誤提示、資料持久化。
完成 MVP 主要使用者流程。
Stage Gate：MVP 可完整跑通核心流程 + 基本測試通過。

Stage 6：測試、優化與部署準備（The Polish）

按照 testing-strategy.md 執行單元測試、整合測試、E2E 測試。
進行效能優化、無障礙檢查（a11y）、SEO 基礎設定。
準備部署流程。
Stage Gate：測試全部通過 + staging 環境部署成功。

Stage 7：維護與迭代（The Evolution）

定義新需求處理流程（更新 prd.md → tasks.md → tech-stack.md → 新 feature branch）。
定期 code review 與重構。
目標：讓專案可以長期穩定共同開發。


三、關於臨時新增插件或 API 的處理規則（已納入）

允許新增，但必須走規範流程，不得直接亂加。
新增時必須：
先更新 tech-stack.md（新增套件名稱、版本、用途、限制）。
更新 tasks.md（新增相關安裝與整合任務）。
更新 decision-log.md（記錄新增理由）。
若影響較大（例如新增 i18n 會影響許多元件），可能需回到 Stage 2 補元件，或開獨立分支處理。
必須經過人類確認後才能安裝與整合。

在 agent-protocols.md 中會加入明確禁止條款：「未更新 tech-stack.md 並獲得人類確認前，禁止安裝任何新套件。」


這個版本已經非常完整：

有明確的任務追蹤（tasks.md）
有彈性處理不確定技術（tech-stack 待確認 + 規範新增流程）
有防鬼打牆機制（agent-protocols + Stage Gate + 強制更新 tasks）
有完整閉環（從 0 到 7）
