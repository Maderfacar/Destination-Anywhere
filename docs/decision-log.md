# 決策紀錄 (Decision Log)

本文件用來記錄專案中重要的技術決策、設計選擇、套件新增原因等。  
每次做出重大決策或新增套件時，Agent 必須立即更新此檔案。

## 決策記錄格式

每次新增紀錄時，請依照以下格式：

**日期**：YYYY/MM/DD  
**決策類型**：技術選擇 / 套件新增 / 設計調整 / 其他  
**標題**：簡短描述  
**背景 / 原因**：為什麼要做這個決策  
**決定內容**：具體做了什麼  
**影響範圍**：會影響哪些部分  
**替代方案**：當時考慮的其他方案（可選）  
**相關文件**：參考了哪些 MD 文件

## 已記錄的決策

### 2026/04/22 - 專案初始化決策

**決策類型**：技術棧選擇  
**標題**：選擇 Nuxt 3 + Vue 3 + Firebase + LINE LIFF 作為主要技術棧  
**背景 / 原因**：  
- 需要快速開發包含三端（乘客、司機、管理者）的訂車平台  
- 需要良好的 LINE 整合與即時通知功能  
- 希望使用 Nitro 作為 BFF 避免金鑰外洩  
- 行動版優先且需要 Dark Mode 支援  

**決定內容**：  
- 前端框架：Nuxt 3 + Vue 3 Composition API + TypeScript（Strict Mode）  
- 樣式：Tailwind CSS + Google Stitch design.md  
- 後端：Nuxt Nitro（server/api/）  
- 資料庫與認證：Firebase Firestore + Firebase Auth  
- 登入方式：LINE LIFF  
- 多語系：繁體中文（預設）+ 英文  

**影響範圍**：整個專案技術架構  
**相關文件**：tech-stack.md、prd.md

### 2026/04/22 - 設計系統選擇

**決策類型**：設計規範  
**標題**：採用 Google Stitch 產出的 Editorial Horizon 設計系統  
**背景 / 原因**：需要高級、通透、現代的視覺風格，符合機場接送服務的專業形象  
**決定內容**：所有 UI 元件與樣式必須嚴格遵循 design.md 與 style-guide.md  
**影響範圍**：所有前端元件與頁面  
**相關文件**：style-guide.md、design.md

### 2026/04/22 - 開發流程與規則制定

**決策類型**：開發流程  
**標題**：建立完整的 MD 規則系統與 Agent 協作協議  
**背景 / 原因**：防止 Agent 鬼打牆、上下文漂移、亂加套件  
**決定內容**：建立 prd.md、tech-stack.md、agent-protocols.md、.windsurfrules 等核心規則文件
