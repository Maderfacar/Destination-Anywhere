# 開發 Roadmap（階段性開發計劃）

本文件定義本專案從 0 到 100 的完整開發階段與每個階段的目標、任務範圍、完成標準。  
Agent 必須**嚴格按照本文件順序**執行，不得跨階段跳躍開發。

**重要提醒**：雖然主要由 Agent 撰寫程式碼，但每個 Stage 仍需包含「寫碼 → 測試 → 修正 → 人類驗證」的循環，因此需要較多時間。

---

## Stage 0：需求與規則最終確認（The Constitution）
**目標**：在寫任何一行程式碼之前，先把專案的「法律」全部立好。

**主要任務**：
- 完成並確認所有核心 MD 文件
- 進行文件一致性檢查
- 產生 tasks.md 完整任務清單

**完成標準**：所有 MD 文件經人類確認無誤
**預計耗時**：1～2 天（主要由人類審核）

---

## Stage 1：環境初始化與規則設定（The Law）
**目標**：建立可正常運行的開發環境。

**主要任務**：
- 建立 Nuxt 3 專案與基礎套件
- 設定 Git、Tailwind、TypeScript、Firebase 等
- 建立所有規則檔案

**完成標準**：執行 `pnpm dev` 可正常看到 Nuxt 預設頁面
**預計耗時**：2～3 天

---

## Stage 2：基礎原子組件開發（The Atoms）
**目標**：建立統一、美觀的基礎 UI 元件。

**主要任務**：
- 開發 Button、Input、Card、Modal 等原子元件
- 所有元件支援 Light / Dark Mode 並嚴格遵循 design.md

**完成標準**：元件展示頁面樣式正確，build 通過
**預計耗時**：5～8 天（UI 細節調整較耗時）

---

## Stage 3：佈局與靜態頁面（The Shell）
**目標**：搭建完整的網站骨架。

**主要任務**：
- 建立 Header、Footer、Layout
- 建立乘客、司機、管理者三端頁面框架

**完成標準**：可正常切換頁面，佈局在手機上顯示良好
**預計耗時**：5～7 天

---

## Stage 4：邏輯實作與狀態管理（The Logic）
**目標**：讓網站開始有互動功能。

**主要任務**：
- LINE LIFF 登入
- 訂單表單邏輯與 Google Maps 計價
- Pinia 狀態管理

**完成標準**：表單可正常操作，計價功能正確
**預計耗時**：7～10 天

---

## Stage 5：資料串接與持久化（The Soul）
**目標**：完成 MVP 核心功能。

**主要任務**：
- Nitro 後端 API
- Firebase 資料儲存
- 航空 API 與 LINE Bot 通知
- 司機位置地圖顯示

**完成標準**：可完整走完訂車到通知的主要流程
**預計耗時**：8～12 天（外部 API 整合最容易卡關）

---

## Stage 6：測試、優化與部署準備（The Polish）
**目標**：提升品質並準備上線。

**主要任務**：
- 測試與除錯
- 行動版優化
- 部署至 Vercel

**完成標準**：測試通過並可成功部署
**預計耗時**：4～6 天

---

## Stage 7：維護與迭代（The Evolution）
**目標**：建立長期維護機制。

**主要任務**：
- 新需求處理流程
- 定期重構

**完成標準**：具備清晰的迭代 SOP
**預計耗時**：持續進行

---

**重要規則**
- 嚴格循序執行，不可跳階段
- 每個 Stage 結束必須通過 Stage Gate（build + lint + 人類確認）
- 若開發速度比預期快很多，可適度壓縮時間；若頻繁鬼打牆，則需延長
